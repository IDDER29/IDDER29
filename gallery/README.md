# 20 README Variants

Each one is a different philosophy. All grounded in the same research finding: **hard-to-fake proof-of-work + restraint beats decoration**. Zero skill percentage bars, zero badge soup, zero vanity widgets in any of them.

Pick one. Or steal the best parts from several.

---

## Research Foundation

These are all built on one core finding: **the widgets everyone uses are cheap to fake, so they signal nothing.** The only things that signal real capability are:

1. **Real artifacts** — shipped products, reviewed code, measurable outcomes
2. **Demonstrated thinking** — decisions explained, trade-offs named, problems scoped
3. **Above-the-fold clarity** — 5 seconds is what you get; use them

Zero of the 20 variants use: streak counters · stat cards · skill bars · badge walls · typing animations · trophy displays · visitor counters.

---

## How to Pick

- Want to stand out visually → **20** (typographic), **03** (editorial), **18** (debug)
- Want to stand out intellectually → **06** (case study), **13** (systems architect), **08** (research lab)
- Want extreme restraint → **14** (one line), **17** (haiku), **01** (minimalist)
- Want personality → **10** (zine), **04** (anti-readme), **11** (narrative)
- Want to convert clients → **12** (product landing), **05** (proof of work), **07** (founder)
