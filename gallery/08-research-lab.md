## Abstract

This profile documents an ongoing investigation into the problem of building software that works — not merely software that compiles. The research spans four primary domains: low-level systems (C), frontend interface engineering (React/Next.js), commerce systems (Shopify), and cross-platform mobile (React Native). The central thesis is that correctness, performance, and craft are not orthogonal concerns but aspects of a single engineering discipline.

## Publications / Shipped Work

### Arpio Architects Platform (2024)
*Case study in conversion-driven interface engineering for professional services.*

**Problem:** Architecture firms rely on trust and aesthetic credibility to acquire clients. Generic web templates fail to establish either. **Method:** Brand-first design system, structured case study format, conversion-optimised contact flow. **Results:** Measurable increase in qualified inquiries within 30 days of deployment. **Stack:** React · TypeScript · Vite · [Repository](https://github.com/IDDER29/arpio-architects-case-study)

### Social Compass (2024)
*Campus super-application: events discovery, ticketing, and community systems.*

**Problem:** Campus event discovery is fragmented across WhatsApp groups, flyers, and word of mouth. Ticketing is manual. Community has no digital layer. **Method:** Unified React Native application with integrated ticketing, reward point accumulation, and community feed. **Results:** Production deployment, active user base. **Stack:** React Native · TypeScript · [Repository](https://github.com/IDDER29/social-compass-mobile)

### 1337 Systems Suite (2023–2024)
*Reconstruction of POSIX standard library components from first principles.*

**Motivation:** The hypothesis that understanding a system's foundation improves one's ability to build on top of it. **Scope:** `printf` implementation (variadic argument parsing, format specifiers); sorting engine (algorithm selection under constraints); `libft` (memory management, string operations, linked lists); `get_next_line` (stateful file descriptor reading); `minishell` (POSIX-compatible shell with pipes, redirections, signal handling). **Finding:** Hypothesis confirmed. **Stack:** C · Makefile · [Repository](https://github.com/IDDER29?tab=repositories&language=c)
